Clumpy

Clumpy is designed to make new m3u playlists to clump together the songs that share words in their filenames. Make your playlists more clumpy with The King of Clump!

Clumpy needs a pre-existing playlist to use as input. You can create one using a command line like this on Linux:

ls *.mp3 *.ogg *.wav *.webm *.m4a *.flac 2>/dev/null | sort -R > playlist.m3u

That creates a shuffled playlist of audio files in the current directory, so cd to the appropriate folder first.

The sorting of the input playlist doesn't matter much to Clumpy because by default, it shuffles the order of loaded songs before starting for the purpose of avoiding having recurring order of songs in the output playlists that are generated.

One problem with clumping is that all songs by the same artist tend to be clumped together if the artist is in the filename. And songs will tend to clump in much the same way again and again, probably not good for any station or stream. To reduce that, simply run for less time to result in less clumping, retaining some of the randomness in the ordering.

Please note that you might need to disable shuffling in your media player in order to notice clumping in the playlists. With VLC, there is a button at the bottom with crossed arrows, or avoid using the -Z option for cvlc.

The program overwrites any old playlist with the same name as the output file.

Clumpy is tiny, free, and open source.

Command Line Options

-iinfile
-ooutfile
-ttime in seconds to run, default 10, more=clumpier
-rshuffle amount, default 8, prevents recurring order
-mllow number of mutations, default 1
-mhhigh number of mutations, default 2
-bbuffer size, default 1024, increase for longer names
-vverbosity, 0 or 1
-hshow this help
